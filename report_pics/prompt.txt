\chapter{Abstract}

{\large
This project proposes the design and implementation of a High QPS (Queries Per Second) Text Search Engine using Apache Solr and ZooKeeper, aimed at efficiently indexing and retrieving large collections of news articles. The system will support keyword-based full-text search with relevance ranking using BM25, along with advanced search capabilities such as faceted filtering, metadata-based querying, and support for rich document formats including HTML and PDF. The goal of the project is to evaluate how modern distributed search infrastructures can handle large query workloads while maintaining low response latency and high throughput.
}

\vspace{10pt}

{\large
To study performance under heavy workloads, the search system will be benchmarked using the Siege load testing tool to simulate concurrent users. The experiments will analyze system behaviour as query load increases and will measure important metrics such as Queries Per Second (QPS), response latency, throughput, and error rate. Through these experiments, the project aims to understand how indexing strategies, query preprocessing, caching, and distributed architectures influence scalability and system performance.
}

\vspace{10pt}

{\large
In addition to the core search engine implementation, the system will include several supporting components to improve usability and monitoring. These include a web-based interface for submitting queries and visualizing results, query preprocessing techniques such as tokenization and stop-word removal, and analytics logging to record search patterns and system response times. These analytics will allow us to identify frequently searched keywords, detect slow queries, and understand workload patterns during high-QPS benchmarking experiments.
}

\vspace{10pt}

{\large
The system will also explore intelligent search assistance mechanisms to improve the user search experience. Auto-suggestion features will provide real-time query completion using prefix-based indexing and Solr suggester components, enabling users to discover relevant queries quickly while typing. These features are considered exploratory extensions intended to enhance search quality and usability.
}

\vspace{10pt}

{\large
As additional exploratory features, the project may incorporate components such as Redis-based query caching for frequently accessed searches, a lightweight ETL pipeline for automated dataset ingestion and cleaning, and a monitoring framework for observing system metrics during benchmarking. A React-based dashboard may also be developed to visualize query patterns, query frequency heatmaps, and server performance metrics through interactive charts.
}

\vspace{10pt}

{\large
Overall, the project demonstrates how scalable search engines can be constructed using distributed database and indexing technologies. By combining indexing, caching, distributed deployment, intelligent query assistance, and performance benchmarking, the system aims to provide insights into the design of high-performance search infrastructures capable of supporting large numbers of concurrent queries.
}

\chapter{Features to be Implemented}

\section{Core Search Engine Features}

\begin{itemize}[leftmargin=*]

\item \textbf{Keyword-based full-text search} across news titles, content, metadata with relevance ranking using BM25

\item \textbf{High-QPS benchmarking} targeting 1000+ concurrent queries/sec using Siege load testing

\item \textbf{Optimized Solr schema} with fields: title, body, date, source, entities, topic facets

\item \textbf{Web interface} for search input/output and benchmark result visualization

\item \textbf{Query preprocessing and normalization}
\begin{itemize}
\item Tokenization and stop-word removal
\item Lowercasing and punctuation normalization
\item Query expansion experiments for improved search results
\end{itemize}

\item \textbf{Faceted filtering}
\begin{itemize}
\item Filtering search results based on metadata fields
\item Example facets: publication date, topic category, news source
\item Enables efficient exploration of large news datasets
\end{itemize}

\end{itemize}

\section{Security Middleware Features}

\begin{itemize}[leftmargin=*]

\item \textbf{Rate limiting} using nginx/Solr plugins to prevent DoS during high-QPS tests

\item \textbf{Input sanitization} against SQL injection/XSS using Solr query parsers

\item \textbf{JWT/OAuth authentication} for admin benchmarking dashboard access

\item \textbf{Audit logging} of high-load queries with anomaly detection

\end{itemize}

\section{UI/Heatmap Dashboard Features}

\begin{itemize}[leftmargin=*]

\item \textbf{Interactive React.js dashboard} with real-time search bar and filters

\item \textbf{Heatmap visualizations} of query frequency by time/topic using D3.js/Leaflet

\item \textbf{Live QPS monitoring} gauges and historical trend charts via Chart.js

\item \textbf{Exportable benchmark reports} as CSV/PDF with embedded graphs

\end{itemize}

\section{Query Analytics and Logging}

\begin{itemize}[leftmargin=*]

\item Logging all incoming search queries with timestamps

\item Measuring response latency for each query

\item Identifying most frequent search terms

\item Detecting slow queries and potential performance bottlenecks

\item Analysis of query distribution during high-QPS benchmarking experiments

\end{itemize}
