\chapter{Abstract}

This project implements a High QPS (Queries Per Second) Text Search Engine using Apache Solr and ZooKeeper, specifically designed for indexing and searching large collections of news articles. The system supports full-text search with relevance ranking, faceted filtering, and rich document handling (HTML/PDF). Key achievements include achieving 1000+ QPS under high load using Siege benchmarking, implementing comprehensive security middleware (rate limiting, authentication, audit logging), and developing an interactive web UI with heatmap visualizations of query patterns and server load. 

The system demonstrates enterprise-grade scalability through SolrCloud multi-node deployment and provides production-ready features like ETL pipelines, query caching, query preprocessing, analytics logging, and performance profiling. The project evaluates distributed search system behaviour under heavy query loads and studies how indexing, caching, and distributed architectures improve scalability.

This project proposes the design and implementation of a High QPS (Queries Per Second) Text Search Engine using Apache Solr and ZooKeeper, aimed at efficiently indexing and retrieving large collections of news articles. Overall, the project demonstrates how scalable search engines can be constructed using distributed database and indexing technologies.
